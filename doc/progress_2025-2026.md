# monoruby 進捗まとめ (2025年4月〜2026年4月)

この一年間（2025年4月〜2026年4月）でmonorubyは約500コミットにわたって大きく進化しました。

---

## 概要

| 期間 | 主なテーマ |
|------|-----------|
| 2025年4〜6月 | 標準ライブラリの拡充、Onigmo正規表現エンジン導入 |
| 2025年7〜9月 | キーワード引数の完全対応、ハッシュ/Enumerable強化、JITリファクタリング |
| 2025年10〜12月 | JIT抽象解釈の改善、最適化、ruby-benchサポート |
| 2026年1〜2月 | FFI/Fiddle対応、継続フレーム導入、各種最適化 |
| 2026年3〜4月 | ruby/spec対応の大規模改善、クラッシュ・パニック修正 |

## ruby/spec 互換性の改善

### 主な改善箇所（2026年2月以降）

ruby/spec core テストスイートに対する大規模な互換性向上が行われました。

| 指標 | 改善前（2026年2月頃） | 改善後（2026年4月） |
|------|----------------------|-------------------|
| F+E（Fail + Error） | ~14,000 | ~10,300 |
| パス率 | ~60% | ~73.6% |
| クラッシュ | 多数 | 0〜数件 |
| cargo test | 568 pass / 10 fail | 756 pass / 0 fail |

### 修正された主なカテゴリ

- **暗黙型変換** (`to_int`/`to_str`/`to_f`/`to_ary`/`to_hash`) の全面整備
- **coerce プロトコル** — 数値演算での型不一致時の変換
- **TypeError/ArgumentError メッセージ** の CRuby 互換化
- **Errno 例外** — `RuntimeError` → 適切な `Errno::*` への変換
- **IOError** の正確な発生
- **Float のフォーマット** (`inf`, `-0.0`, 科学的記数法)
- **frozen オブジェクト** — JIT でのフリーズチェック追加
- **メソッドアリティ** — 30以上のビルトインメソッドのアリティ修正 (#243, #245)
- **Integer** 仕様 — 10以上のカテゴリで修正 (#263, #265, #268, #270, #272)
- **Rational** の完全な再実装 (#266)

## まとめ

この一年間で monoruby は：

1. **正規表現エンジン（Onigmo）の統合** による本格的な正規表現サポート
2. **FFI/Fiddle/SQLite3** 対応による外部ライブラリ連携
3. **数百のビルトインメソッド** の追加・修正
4. **ruby/spec 通過率を約60%から73.6%に向上**（Fail+Error を14,000→10,300に削減）
5. **JITコンパイラの継続的な改善** — 不動点反復、定数関数、継続フレームなどの新機能
6. **クラッシュ・パニックをゼロに近い状態** へ改善（`cargo test` は756 pass / 0 fail達成）

という大きな進歩を遂げました。
